# UI

This folder contains the early UI direction for SkillSpring Quantum.

The intended end state is:

- a desktop-first local application
- a lightweight browser extension that acts as a helper surface

## Product stance

The desktop app is the main operating environment.
The extension is a light intake and handoff layer.

## Current contents

- docs/ -> UI planning and screen definitions
- prototype/ -> lightweight static prototype shell
- extension/ -> early extension direction
- shared/ -> navigation and design tokens
- app/ -> desktop app scaffold
- app-shell/ -> shell architecture and command bridge planning

## Key principle

The UI should map to real backend workflows.
No fake approval model, no invented data lifecycle, no decorative control plane.
