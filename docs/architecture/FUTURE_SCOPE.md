# Future Scope

This document records the main expansion areas that are intentionally outside the current MVP center of gravity.

## Current stance

Quantum's current promise stays narrow:

- import major AI conversation exports locally
- produce readable archive output
- produce privacy-aware dataset artifacts
- explain trust, fallback, and partial-import behavior in plain language

Future-scope work should not displace that loop before private beta.

## Likely expansion areas

- broader normalization adapters for additional source formats
- richer intent and metadata extraction
- secondary AI vendor expansion beyond the current marketed set
- stronger domain extraction and cross-reference logic
- more advanced privacy and entity detection
- configurable grouping strategies across threads and domains
- batch orchestration and performance controls for larger corpora
- broader document ingestion as a deliberate product expansion, not accidental scope drift

## Decision rule

When considering future-scope work, ask:

- does it improve the current export -> archive -> dataset workflow more than the next beta-readiness slice?
- does it help trust, recovery, search, archive review, or dataset review now?
- is it more important than packaging, tester support, or confirmed blocker fixes?

If not, defer it.
