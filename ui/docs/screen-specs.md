# Screen Specs

## 1. Dashboard
Purpose:
- show health of the whole system at a glance

Widgets:
- last batch run status
- conversations processed
- duplicates skipped
- purge rate
- private review rate
- review queue size
- curated promotions count
- recent warnings

## 2. Imports
Purpose:
- drag and drop export folders or files
- start single or batch runs

Actions:
- choose input folder
- choose output root
- run batch
- view live run log

## 3. Organized Output
Purpose:
- browse topic folders and markdown snippets

Actions:
- open topic folder
- preview markdown
- move to purge
- restore from purge
- inspect manifest linkage

## 4. Datasets
Purpose:
- inspect versioned datasets and manifests

Actions:
- filter by topic
- filter by signal tier
- inspect run manifests
- export subset later

## 5. Tiered DB
Purpose:
- inspect raw, processed, curated, and private review records

Actions:
- browse by tier
- inspect record metadata
- view fingerprint status
- review manifests

## 6. Review Queue
Purpose:
- inspect near-curated records

Actions:
- approve for curated
- reject
- tag for later review
- inspect reasons

## 7. Diagnostics
Purpose:
- inspect latest runs and batch history

Actions:
- open latest run
- compare batch deltas
- inspect failures
- inspect recommendations

## 8. Governance
Purpose:
- inspect and edit rule files

Actions:
- edit thresholds
- edit topic filter rules
- edit curation rules
- inspect governance logs
