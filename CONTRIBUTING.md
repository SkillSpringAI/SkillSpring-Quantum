# Contributing

## Audience

This document is for contributors and developers.

## Before opening a change

Read these first:

- [docs/README.md](docs/README.md)
- [Architecture](docs/technical/ARCHITECTURE.md)
- [Development Guide](docs/technical/DEVELOPMENT_GUIDE.md)
- [Testing](docs/technical/TESTING.md)
- [MVP Roadmap](docs/project/MVP_ROADMAP.md)

## Contribution expectations

- Keep the product story centered on the local import -> archive -> search -> dataset workflow
- Prefer one authoritative document for each topic
- Avoid widening the README into a technical reference
- Keep user-facing language product-first and outcome-first
- Add or update tests when behavior changes

## Scope discipline

Quantum is preparing for private beta. Changes that improve installation, import trust, archive review, dataset review, retrieval, diagnostics, or documentation clarity should take priority over speculative expansion.

## Pull request notes

- Describe the user-facing change
- Call out any documentation updates
- Mention test coverage or why tests were not run
