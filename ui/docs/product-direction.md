# Product Direction

## Primary surface
SkillSpring Quantum should function primarily as a desktop application.

This is where users should:
- import conversation exports
- run processing workflows
- inspect organized outputs
- browse datasets
- manage tiered DB records
- review diagnostics
- manage purge and restore decisions
- operate review queue and curated promotion

## Secondary surface
A lightweight extension may later support:
- quick send to desktop app
- quick mark/save/bookmark
- basic context capture
- launch desktop app from browser context

The extension should not become the main processing environment.

## Design principle
Desktop app = control plane
Extension = intake utility
